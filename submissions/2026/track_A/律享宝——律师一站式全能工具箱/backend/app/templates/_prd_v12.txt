3.1 法律文书模板库总规则

数据保存：用户填写的表单信息自动保存至云端，支持中途退出后继续编辑
模板更新：预留模板后台管理接口，支持后续新增、修改、删除模板，无需修改前端代码

四、交互设计说明
模板列表页：
左侧显示分类导航，右侧显示对应模板卡片，卡片包含模板名称、依据来源、适用场景
顶部搜索框支持模糊搜索，输入关键词自动匹配模板名称和内容
每个模板卡片右下角有 "立即生成" 按钮，点击进入填写页
表单填写页：
表单按官方模板的结构分块展示，每块有明确的标题
必填项用红色 * 标注，未填写时无法点击 "生成预览"
勾选框支持点击切换选中 / 未选中状态，生成的文书中显示为 "☑" 或 "□"
大文本输入框（如事实与理由）支持富文本编辑，保留换行和分段
右侧显示实时预览窗口，填写内容实时同步到预览中
底部有 "保存草稿"、"生成预览"、"返回" 三个按钮
预览导出页：
预览窗口显示完整的文书样式，支持放大、缩小、翻页
底部有 "导出 Word"、"导出 PDF"、"返回编辑"、"保存至个人中心" 四个按钮
导出的 Word 文档保留所有格式和可编辑性，PDF 文档添加平台水印（可选）
五、技术实现要求
模板引擎：使用 FreeMarker 或 Thymeleaf 作为模板引擎，将官方模板转换为.ftl 或.html 模板文件，用 ${字段名} 作为占位符
表格处理：所有表格使用 HTML 表格实现，确保在 Word 和 PDF 中格式不变，合并单元格、边框样式与官方模板一致
导出功能：
Word 导出：使用 POI 或 LibreOffice 实现，保留字体、字号、页边距、表格样式
PDF 导出：使用 iText 或 Flying Saucer 实现，支持添加水印和加密
数据存储：用户填写的表单数据以 JSON 格式存储在 MySQL 数据库中，关联用户 ID 和模板 ID
权限控制：所有模板对所有用户开放，导出功能无需登录即可使用（登录后可保存草稿和历史记录）
接口设计：
GET /api/templates：获取模板列表和分类
GET /api/templates/{id}：获取指定模板的表单结构
POST /api/templates/{id}/generate：生成文书预览
POST /api/templates/{id}/export：导出文书
POST /api/drafts/save：保存草稿
GET /api/drafts：获取用户草稿列表
